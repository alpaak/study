五层：
        应用层：开发工程师   PDU(协议数据单元)
            应用层协议 = 传输层协议 + 端口号
            TCP(传输控制协议) + 23端口 = telnet协议(远程控制协议)
            TCP + 80/443端口 = web协议
            UDP(用户数据协议) + 53 = DNS协议（域名解析）

        传输层：安全工程师  Segment(数据段)
            有两个协议：TCP(比较靠谱)  UDP(不靠谱)
            端口号（16bit）：代表一个主机的不同服务
               范围：0~65535（2的16次方）（0~1023是知名端口，1024~65535是用户可以使用的）
                源端口号（Sport）：一般源端口号是随机产生的
                目的端口号（Dport）: 一般是一些知名端口

            80端口：web服务
            21端口：FTP服务
            22端口：ssh服务
            23端口：telnet服务

        网络层：网络工程师  Packet(数据包)
            IP地址：三层逻辑地址，是可变的，网络设备主要靠IP地址进行访问
                源IP(SIP)：
                目的IP(DIP)：
                一般在数据包的传输过程中，SIP和DIP是不变的,
                如果有 NAT，也是会变化的

        链路层：网络工程师   Frame(数据帧)
            MAC地址（48bit）：二层物理地址，指的就是每一个网络设备生产之后就会有唯一的MAC地址，可以全球唯一标识网络设备
                前24bit是组织分给生产商的，后24bit是生产商自行定义的，
                源MAC（SMAC）：
                目的MAC（DMAC）：
                一般在数据的传输过程中，SMAC和DMAC是不断变化的
                跨网段的时候，DMAC是会变化的，会经过多次的网关

        物理层：  Bit流
            网线（双绞线）（光纤）：通过二进制传输


    通过一个公网的IP 访问server的技术，一般使用的是 NATServer 技术

    SW（交换机）, FW（防火墙）, R（路由）
